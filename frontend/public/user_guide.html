📖 Platform Overview

Welcome to the ITACS Enterprise Insights Platform—the strategic multi-agent workbench built to accelerate functional oncology slide deck synthesis, cross-functional wargaming, and boardroom-certified action formulation.

ITACS operates using a **decoupled, multi-agent consensus network** designed to eliminate strategic bias, evaluate evidence levels, and ground corporate decisions (for assets like V940 and MK-1084) in solid, verifiable data. The platform ensures 100% compliance with FDA regulations and brand guidelines by routing extracted slide insights through automated supervisor gates, challenging assumptions via Red Team agents, and compiling a cryptographically secured corporate audit trail.

💡
Did you know? ITACS leverages a hybrid routing engine built on **Google GenAI SDK**: using **Gemini 1.5 Pro** for deep strategic wargaming and RAG search, and **Gemini 2.0 Flash** for sub-second slide parsing, compliance checks, and real-time UI rendering.

🖥️ Interface & UI Guide

ITACS is structured as a **Single Pane of Glass** to eliminate "format fatigue" and abstract complex backend agent interactions into a clean, intuitive canvas. Below is a detailed breakdown of each tab, panel, and page in the workspace, along with the business and compliance rationale for why it exists.

1. The Four Main Workspace Tabs

The top navigation allows you to switch between different operational dimensions of the ITACS platform:

Tab Page What It Displays Why It Exists (Business Value)
Indication Roadmap The primary strategic timeline containing clinical studies, filing pipelines, and active regulatory milestones. Provides a single visual workspace for leaders to track asset launch schedules (V940, MK-1084) without bouncing between tools.
SME Validation Queue The active incoming queue of compliance-cleared slide insights waiting for functional expert validation. Acts as the critical **Human-in-the-Loop gate** where clinical researchers confirm, edit, or quarantine AI-extracted findings.
Strategic Builder The cross-functional Kanban workspace where validated insights are formulated into macro-imperatives. Guarantees that all corporate strategic initiatives are **directly grounded in source evidence cards** via the actions ledger.
Memory Audit Ledger A chronological, cryptographic ledger recording every user edit, validation, and SHA-256 digital seal. Provides an **immutable audit trail** for legal and regulatory compliance reviews, proving data provenance.

The main cockpit is split into three highly focused columns to maximize strategic focus and operational visibility:

📥 Workflow 1: Ingestion & Validation Core

This workflow covers uploading raw slide decks, checking compliance, and approving insights into the active memory bank.

1

Upload the Slide Deck

Drag and drop your functional PowerPoint/PDF file (e.g., Melanoma_MR_Brief_Q2.pptx) into the upload zone.

2

Audit Compliance & Historian Badging

The system will parse the slide, run compliance audits, and flag the insight as 🆕 New or 🔄 Known based on vector similarity.

3

Approve to Enterprise Memory

Open the card details, review the evidence score and RAG quotes, and click **"Approve to Memory Bank"** to write the card to the database.

🔍 How to Validate the Results

  • Confirm the newly validated card displays a solid emerald-green checkmark and is listed under the validated ledger.
  • Verify the total counts in the top header update immediately.

⚖️ Workflow 2: Strategic Workshop & Actions Core

This workflow covers formulating macro-imperatives, cataloging trade-offs/risks, and adding actionable tactics.

1

Formulate a Strategic Imperative

Navigate to the **Strategic Builder** page. Click **"+ Formulate Imperative"**, fill the title/description, and create the card on the board.

2

Catalog Resource Trade-offs & Risks

Click the card to slide open the Strategic Workshop Drawer. In the Implications Editor, type the trade-offs and risks. Click out of the fields to trigger automatic on-blur saving.

3

Add Actionable Tactics

In the Actions Ledger at the bottom of the drawer, type your tactical action, select the owner, link a validated source card as evidence, and click **"Add"**.

🔍 How to Validate the Results

  • Verify the Strategic Workshop Drawer remains open during typing (no focus loss or text wipes).
  • Confirm the strategic card in the background dynamically updates its count to 📋 1 Tactical Actions.

🔍 Workflow 3: Gap Detection & Temporal Decay Advanced

To prevent strategic drift, ITACS features an active **Temporal Decay Engine** that flags cards older than 90 days as stale, automatically querying external medical databases to fill gaps.

1

Trigger Gap Detection

Click the **"Automated Gap Engine"** button in the sidebar or run the API endpoint to scan for stale validated cards.

2

External Repository Search (MCP)

The model utilizes the Model Context Protocol (MCP) to query registered external endpoints (such as Veeva Promomats or SharePoint) for fresh clinical findings.

3

Merge Gap-Fill Hypothesis

Review the automatically generated gap-fill hypothesis card, and click **"Approve & Merge"** to overwrite the stale card in memory.

🔧 Troubleshooting, Diagnostics & Triage

To establish operational maturity and prevent IT escalation bottlenecks, strategic support teams can run basic diagnostic workflows on local and cloud runtimes.

A. Deciphering Simulated Gateway Logs

Because ITACS runs a simulated zero-trust OIDC gateway, it is programmed to mock network timeouts. If you see the following logs in your console, do not panic:

[Resiliency Loop] L2_Oidc_Ingestion_Agent call failed (Attempt 1/3): Gateway Timeout (504).
[Resiliency Loop] L2_Oidc_Ingestion_Agent call failed (Attempt 2/3): Gateway Timeout (504).
[Resiliency Loop] Network timeout. Routing to local Skeptical Challenger cache.

Diagnosis: The simulated network timeout kicked in. Notice that the pipeline did not crash; it safely engaged the skeptical challenger local fallback cache to present the strategic debate card.

B. Diagnostic Capture Checklist

Before submitting a ticket, run these steps to gather diagnostics:

  • Service Port Check: Verify FastAPI (port 8000) and React (port 3000) are running: lsof -i :3000,8000.
  • Audit Logs: Copy the full log trace from the bottom of the left sidebar. It includes critical execution time metrics.
  • State Check: Note the active sub-agent in the wargaming scanner pipeline to pinpoint the step that timed out.

📜 Cryptographic Audit Ledger & Data Provenance

To satisfy rigorous pharmaceutical legal and FDA audits, ITACS provides cryptographic proof of content integrity. This ensures that approved strategic insights are locked and cannot be modified after validation.

A. How the SHA-256 Seal Works

When an oncology SME validates an insight, the system binds the text to its original reference code and generates a secure, unique **SHA-256 verification hash**:

{
  "card_id": "9b1deb4d-3b7d-4bad-9bdd-2b0d7b3d4bad",
  "evidence_source": "Melanoma Slide 14",
  "cryptographic_seal": "sha256:e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
}

This hash acts as a **tamper-evident digital wax seal**. If any external user attempts to modify a single character in the database, the computed hash will break instantly, flagging the card as ungrounded and disabling downstream tactical exports.

B. Verifying Immutability in the Audit Ledger

To audit any card's history, navigate to the **Memory Audit Ledger** page. Each entry displays a timestamp, user role, claims verified, and the generated hash. If the hash matches your exported brief, the content is 100% genuine and grounded.

👤 User Personas & Identity Governance

Oncology brand compliance is highly regional and role-dependent. ITACS enforces **Zero-Trust Identity Governance** using SPIFFE/OIDC OAuth JWTs, ensuring users only access and modify what their role permits.

A. Core Operational Roles

The platform supports three distinct roles, which can be toggled in the header actions:

  • Oncology SME (e.g. nitinagga-ge-2): Authorized to validate insights in the queue, run wargaming simulations, and approve cards to the Memory Bank.
  • Strategic Lead: Authorized to formulate strategic imperatives on the board, edit implications (trade-offs, risks), and add tactical actions.
  • Regulatory Admin: Authorized to register MCP servers, edit systems architecture diagrams, and view the raw cryptographic database ledger.

B. Role-Based Context Isolation

User roles seamlessly propagate down to the GenAI RAG context. The orchestrator dynamically filters the pgvector memory search based on the active user's permissions, ensuring that unvalidated or pending competitor insights are completely isolated from unauthorized users.
